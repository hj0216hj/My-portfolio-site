나루나루 (NaruNaru)

2025년 3~6월

지역 맛집 추천 모바일 애플리케이션

사용 기술

React Native Expo Node.js MongoDB Kakao Map API

프로젝트 개요

분류: 모바일 애플리케이션 (Android)
개발 인원: 3인 팀 프로젝트 (팀명: 빡김)
개발 기간: 2025년 3월 ~ 6월 (4개월)
담당 역할: 백엔드 개발 및 데이터베이스 설계
개발 환경: Android Studio, React Native/Expo

나루나루는 사용자의 위치 기반으로 주변 맛집을 추천해주는 모바일 애플리케이션입니다. 사용자 참여형 추천 시스템을 통해 신뢰할 수 있는 맛집 정보를 제공하며, 특히 'AI 음식점 추천' 기능을 통해 식사 후 방문할 카페나 장소를 추천해주는 차별화된 서비스를 제공합니다.

개발 배경 및 기획 의도

시장 분석

코로나 이후 외식 산업이 회복세를 보이고 있으며, 단순 위치 정보에서 벗어나 실제 이용자 리뷰, 메뉴 추천, 예약 서비스까지 통합된 서비스에 대한 선호도가 증가하고 있습니다. 또한 SNS 기반 맛집 인증/공유 문화가 정착되어 인스타그램, 유튜브 등이 중요한 맛집 정보 채널로 자리매김했습니다.

문제 인식

  • 기존 맛집 검색 서비스는 광고성 콘텐츠가 많아 신뢰도가 낮음
  • 개인의 취향과 상황에 맞는 맞춤형 추천이 부족
  • 식사 후 다음 행선지(카페, 디저트 등)를 찾기 어려움
  • 신규 개업 음식점의 홍보 채널 부족

나루나루 차별화 포인트

  • 사용자 참여형 맛집 추천 모델 강화
  • AI 음식점 추천 기능 - 주변 카페/디저트 매장 추천
  • 직관적인 UI 설계로 사용자 경험 개선
  • 신규 개업 음식점 타겟 홍보 플랫폼 제공

주요 기능

1. 위치 및 키워드 기반 맛집 검색

  • 현재 위치 또는 검색한 위치 주변의 음식점 검색
  • 음식 종류, 메뉴 이름으로 키워드 필터링
  • 추천순(좋아요, 하트 등)으로 정렬하여 신뢰도 높은 맛집 우선 표시
  • 거리별, 카테고리별 정렬 기능

2. AI 음식점 추천 (핵심 차별화 기능)

  • 회원가입 시 입력한 선호 음식 데이터를 기반으로 추천
  • AI에게 원하는 느낌의 방문할 주변 카페, 디저트 매장 등 질문하는 방식

3. 음식점 키워드 자동 추출

  • API로 수집한 음식점 데이터에서 주 메뉴, 종류 분석
  • 해당 음식점을 대표하는 키워드 자동 생성
  • 검색 정확도 향상 및 추천 알고리즘 개선

4. 사용자 참여형 리뷰 시스템

  • 사용자가 직접 맛집 등록 및 리뷰 작성
  • 좋아요/하트 기반 추천 순위 시스템

5. 지도 기반 시각화

  • Kakao Map API 연동으로 맛집 위치 시각화
  • 주변 음식점 한눈에 확인

SWOT 분석

Strength (강점)

  • 사용자 참여형 모델로 신뢰도 높은 정보 제공
  • 다음 행선지 추천이라는 새로운 기능
  • 직관적인 UI 설계

Weakness (약점)

  • 초기 데이터 부족
  • 위치 정보의 정확성 및 최신 상태 유지 관리 필요
  • 광고 의존적 수익 모델

Opportunity (기회)

  • 코로나 이후 외식 산업 회복세
  • 배달/픽업 서비스 등 연계 서비스 확장 가능성
  • 신규 개업 음식점 타겟 홍보 플랫폼 제공으로 시장 확대

Threat (위협)

  • 사용자의 거짓 리뷰나 광고성 리뷰 문제
  • 지역 음식점과의 지속적인 협력 관계 구축의 어려움
  • 타베로그 등 기존 서비스와의 경쟁

시스템 아키텍처

프론트엔드

React Native + Expo: 크로스 플랫폼 개발 프레임워크를 사용하여 Android 앱을 개발했습니다. Android Studio를 IDE로 사용하여 효율적인 개발 환경을 구축했습니다.

  • React Native로 컴포넌트 기반 UI 구성
  • Expo를 활용한 빠른 개발 및 테스트
  • 직관적인 UI/UX 설계로 사용자 접근성 향상

백엔드

Node.js: 경량화되고 빠른 서버 구축을 위해 Node.js를 선택했습니다. RESTful API 구조로 클라이언트와의 통신을 구현했습니다.

  • 사용자 인증 및 회원 관리 API
  • 맛집 데이터 CRUD API
  • 리뷰 및 평점 관리 API
  • 추천 알고리즘 서버 로직 구현

데이터베이스

MongoDB: NoSQL 데이터베이스를 사용하여 유연한 데이터 구조를 지원합니다.

  • 사용자 정보, 선호도 데이터 저장
  • 맛집 정보, 키워드, 위치 데이터 저장
  • 리뷰 및 평점 데이터 관리
  • MongoDB Atlas를 활용한 클라우드 DB 운영

외부 API

  • Kakao Map API: 지도 표시 및 위치 기반 검색
  • Kakao Local API: 주변 음식점 정보 수집

기술적 도전 과제 및 해결

1. 추천 알고리즘 설계

문제: 사용자의 선호도를 반영한 맛집 추천 알고리즘을 설계하는 것이 복잡했습니다.

해결:

  • 회원가입 시 사용자의 선호 음식 카테고리 수집
  • 사용자의 검색 이력 및 리뷰 데이터 분석
  • 거리, 평점, 사용자 선호도를 종합한 가중치 기반 추천

2. 외부 API 연동 및 데이터 처리

문제: Kakao API에서 받아온 대용량 맛집 데이터를 효율적으로 처리하고 저장하는 것이 과제였습니다.

해결:

  • 필요한 데이터만 선별하여 MongoDB에 저장
  • 키워드 자동 추출 기능으로 검색 효율성 향상
  • MongoDB의 인덱싱 기능을 활용한 빠른 쿼리 성능
  • 비동기 처리로 API 응답 시간 최적화

3. 초기 데이터 및 사용자 확보

문제: 서비스 초기 단계에서 맛집 데이터와 사용자 리뷰가 부족했습니다.

해결:

  • Kakao Local API를 통한 기본 맛집 데이터 수집
  • 맛집 마스터 등급 시스템으로 사용자 참여 유도
